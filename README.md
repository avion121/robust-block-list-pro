# Robust Block List Pro

Robust Block List Pro is a project that consolidates multiple block list sources into a single, comprehensive list. The list is automatically updated via a GitHub Actions workflow and includes header information displaying the total number of unique blocked items and the last update timestamp.

## How It Works

- **Data Collection:**  
  The Python script (`generate_list.py`) fetches block list data from various sources including uBlock filters, EasyList, malware trackers, and more.
  
- **Data Processing:**  
  The script removes duplicate entries and sorts the combined list.
  
- **Header Information:**  
  A header is added at the top of the generated list (`robust_block_list_pro.txt`), which includes:
  - Title and description of the list.
  - Total number of unique blocked items.
  - The date and time (UTC) of the latest update.
  
- **Automation:**  
  A GitHub Actions workflow (`.github/workflows/update_list.yml`) is set up to run the Python script daily at 5 AM UTC (or manually via workflow dispatch), ensuring the block list stays current.

## Repository Structure

- **.github/workflows/update_list.yml**  
  Contains the GitHub Actions configuration that:
  - Checks out the repository.
  - Sets up a Python environment.
  - Installs dependencies.
  - Runs the Python script.
  - Commits and pushes any updates to the block list file.

- **generate_list.py**  
  The Python script that:
  - Fetches data from a list of specified URLs.
  - Consolidates and processes the data.
  - Writes the final output to `robust_block_list_pro.txt` with the appropriate header.

- **robust_block_list_pro.txt**  
  The output file generated by the Python script containing the combined and updated block list.

## How to Use

1. **Clone or Fork the Repository:**

   ```bash
   git clone https://github.com/your-username/robust-block-list-pro.git
   cd robust-block-list-pro
Configure Secrets:

The workflow uses the default GITHUB_TOKEN provided by GitHub Actions, so no additional secrets are needed.

Customize Sources:

To add or remove sources, modify the urls list in generate_list.py.

Automatic Updates:

The GitHub Actions workflow is scheduled to run daily at 5 AM UTC.

You can also trigger a manual update via the GitHub Actions tab in your repository.

Requirements
Python 3.x

Requests library:
The requests library is installed automatically by the workflow.
